Индекс - структура данных позволяющая быстро искать информацию в большой таблице.
Индексы бывают разные, зависят от целей, структуры данных по которым необходимо искать и предназначены для разных вещей.
Индексы занимают определенное место на диске для более удобного поиска информации.
По сути индекс таблица, которая содержит указатель на физическое место на диске, где находятся данные которые нам нужны.
Индексы удобны для поиска информации, но при их введении время записи возрастает.
Не нужно делать индекс на каждое поле таблицы, т.к. в случае добавления новых строк в таблицу может случиться перезапись индексов под текущий объем данных, а это занимает время.

# Hash индекс
![[Pasted image 20251115204503.png]]
Основывается на хэш таблицах, где данные распределяются по хэшам.
## Как работает Hash Index
- через математическую функцию (собственно она и является хэшем) конвертируем имя человека в хэш;
- на выходе получаем хэш-число;
- те строки, у которых хэш одинаковый складываются в один бакет;
- когда нам нужно найти какое-либо значение, высчитывается хэш;
- находим бакет, где лежат строчки;
- бакет - физическая страница на диске, которая обновляется по мере того как добавляются данные;
- находим строчки, где значение столбца равно этому хэшу;

Хэш индекс используется для поиска по точному назначению
Where Column ==
И дальше мы выходим на нужный бакет и ищем нужное нам значение.
Когда в бакете становится слишком много информации может потребоваться перерасчет всех бакетов и тогда происходит перерасчет всех индексов и добавляются новые бакеты.
