# Вводная
Для использования отслеживаемости должны быть веские причины, т.к. отслеживание взаимосвязанности требований довольно затратный ресурс:
- анализ влияния изменения;
- отслеживание источников;
- повторное использование;
- поиск ненужных/пропущенных требований;

# Связи отслеживаемости требований
Связи требований документируют зависимости и логические связи отдельных требований и других элементов системы. В эти элементы входят:
- бизнес-правила;
- требования различных типов;
- архитектура;
- другие компоненты дизайна;
